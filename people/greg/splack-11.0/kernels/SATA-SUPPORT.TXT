The following list shows which kernels support native Serial ATA, or 
SATA.  The ones with '=y' have SATA support built into the kernel, while 
the ones with '=n' would require an initrd in order to support SATA.

The reason all the kernels don't have built-in SATA where it would be 
sense to have it is so that these kernels (except the huge ones) can 
still be booted from floppy disks formatted to the standard 1.44MB.

No matter how modular things are made, soon a kernel that isn't almost 
totally modular will not fit on a floppy disk.  There will be other ways 
to boot, but the old reliable floppy disk era is quickly coming to a 
close.

NOTE:  You can ignore any line that contains "BLK_DEV_IDE_SATA", as 
that's the older SATA support that was replaced by the SCSI_SATA" 
subsystem.  This list was generated with:  grep SATA *.?/config
